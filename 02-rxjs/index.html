INTRO

Intro to RxJS

The JavaScript implementation of ReactiveX — Observable streams, powerful operators, and a clean composable API.

What is RxJS?
  • ReactiveX for JavaScript — part of the ReactiveX family (RxJava, Rx.NET, RxSwift…)
  • Observer Pattern + Iterator Protocol + functional operators
  • One Observable handles: events, HTTP, timers, state
  • Powers Angular's HTTP client, NgRx, and more
Observable Contract
  • next(value) — emit a value
  • error(err) — terminal failure
  • complete() — terminal success

Rules: zero or more next, then either one error or one complete. Nothing after terminal signals.

The pipe() API — Composable Operators
JavaScript 
import { fromEvent, interval } from 'rxjs';
import { map, filter, take } from 'rxjs/operators';

// Nothing starts until .subscribe() is called — Observables are LAZY
const subscription = interval(1000).pipe(
  map(n => n * 2),           // transform each value
  filter(n => n % 4 === 0),  // only pass multiples of 4
  take(5)                    // complete after 5 values
).subscribe({
  next:     value  => console.log('next:', value),
  error:    err    => console.error('error:', err),
  complete: ()     => console.log('done!')
});

// Cancel at any time
subscription.unsubscribe();

share() and shareReplay()
JavaScript 
import { share, shareReplay } from 'rxjs/operators';

// Cold → Hot: share one execution across all subscribers
const shared$ = expensiveHttp$.pipe(share());
shared$.subscribe(userA);  // triggers HTTP once
shared$.subscribe(userB);  // joins the same execution

// Cold → Hot + replay last value to late subscribers
const config$ = loadConfig$.pipe(
  shareReplay({ bufferSize: 1, refCount: true })
);
// late subscriber still gets the last emitted config
config$.subscribe(lateComponent);

OPERATORS

Key Operators

The building blocks you'll use in almost every RxJS pipeline.

Transform
JavaScript 
// map — transform each value
source$.pipe(
  map(x => x * 2)
);
// 1, 2, 3  →  2, 4, 6

// scan — running accumulator (like reduce, but emits each step)
source$.pipe(
  scan((acc, val) => acc + val, 0)
);
// 1, 2, 3  →  1, 3, 6
Filter
JavaScript 
// filter — pass only matching values
source$.pipe(
  filter(x => x % 2 === 0)
);
// 1, 2, 3, 4  →  2, 4

// take — complete after N values
source$.pipe(take(3));

// takeUntil — complete when notifier fires
source$.pipe(takeUntil(destroy$));
Time-based
JavaScript 
// debounceTime — emit only after silence
source$.pipe(
  debounceTime(300)  // wait 300ms of quiet
);

// throttleTime — emit at most once per interval
source$.pipe(
  throttleTime(1000) // max once per second
);

// delay — shift all emissions by a duration
source$.pipe(delay(500));
Deduplication & Combination
JavaScript 
// distinctUntilChanged — skip consecutive duplicates
source$.pipe(distinctUntilChanged());
// 1, 1, 2, 2, 3  →  1, 2, 3

// combineLatest — emit when any source emits (using all latest)
combineLatest([price$, quantity$]).pipe(
  map(([price, qty]) => price * qty)
);

// forkJoin — wait for ALL to complete (like Promise.all)
forkJoin({ user: user$, posts: posts$ })
  .subscribe(({ user, posts }) => render(user, posts));
Error Handling
JavaScript 
source$.pipe(
  catchError(err => {
    console.error(err);
    return of([]);          // fallback value — stream continues
    // or: return EMPTY;    // complete silently
    // or: throwError(err); // re-throw
  }),
  retry(3),               // retry up to 3 times before erroring
  retryWhen(errors =>     // retry with exponential back-off
    errors.pipe(
      delayWhen((_, i) => timer(Math.pow(2, i) * 1000))
    )
  )
);

THE BIG DECISION

Flattening Operators

mergeMap vs switchMap vs concatMap vs exhaustMap — the most important choice in any RxJS pipeline.

What they do: Each maps an outer value to an inner Observable. They differ in how they handle overlap — what happens when a new outer value arrives before the previous inner finishes.

Decision Table
OperatorConcurrencyOn new valueBest for
mergeMap All in parallel Start new, keep all running Parallel HTTP, fire-and-forget tasks
switchMap Only latest Cancel previous, start new Search, autocomplete, live filters
concatMap One at a time (queued) Queue until current completes Sequential saves, ordered writes
exhaustMap First wins Ignore until current completes Login button, form submit guard
Rule of thumb: Default to switchMap for UI interactions. Use concatMap when order matters. Use mergeMap when you want everything in parallel. Use exhaustMap to prevent double-submit.

RxJS + React — Live Search

A real-world autocomplete combining debouncing, deduplication, and stale-request cancellation.

▶

▶

▶

React Component
TypeScript · React 
import { useEffect, useRef, useState } from 'react';
import { fromEvent, from, of } from 'rxjs';
import { map, debounceTime, distinctUntilChanged, switchMap, catchError } from 'rxjs/operators';

interface Result { id: string; name: string; }

export function LiveSearch() {
  const [results, setResults] = useState<Result[]>([]);
  const [loading, setLoading] = useState(false);
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    const input = inputRef.current!;

    const subscription = fromEvent<InputEvent>(input, 'input')
      .pipe(
        map(e => (e.target as HTMLInputElement).value.trim()),
        debounceTime(300),          // wait 300ms after last keystroke
        distinctUntilChanged(),     // skip if same query repeated
        switchMap(query => {        // cancel previous, start new request
          if (!query) return of([]);
          setLoading(true);
          return from(
            fetch(`/api/search?q=${query}`).then(r => r.json())
          ).pipe(
            catchError(() => of([]))  // fallback empty on error
          );
        })
      )
      .subscribe(data => {
        setResults(data);
        setLoading(false);
      });

    return () => subscription.unsubscribe(); // ← always clean up in useEffect!
  }, []);

  return (
    <div>
      <input ref={inputRef} placeholder="Search..." />
      {loading && <span>Searching...</span>}
      <ul>{results.map(r => <li key={r.id}>{r.name}</li>)}</ul>
    </div>
  );
}
debounceTime(300)

Fires only after 300ms of silence. Prevents spamming the API on every keystroke.

distinctUntilChanged

Skips if the query hasn't changed. Avoids duplicate requests for the same term.

switchMap

Cancels the previous in-flight request when a new query arrives. No stale results.
