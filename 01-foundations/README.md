# Part 1 — Foundations

This module provides the foundational boilerplate for covering reactive programming in plain JavaScript.

## Topics

- The Reactive Manifesto
    Systems should be responsive, resilient, elastic and message driven
    In a nutshell, it is a paradigm to code in which allows to work with data flow in a asyncronous approach.
- Observer and Iterator patterns
    Common Design patterns that are reactive
- explain pull vs push delivery
- compare promises and observables
- complete guided boilerplates for core reactive ideas

## Structure

- `slides/` → short theory decks
- `demos/` → consistent starter boilerplates for live coding
- `exercises/` → guided practice with TODOs and solutions
